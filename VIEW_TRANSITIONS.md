# View Transitions Status

Документ фиксирует текущее состояние, чтобы избежать расхождения docs и кода.

## Текущее состояние

- `next-view-transitions` не используется.
- `React ViewTransition` API в проекте не подключен.
- `experimental.viewTransition` в `next.config.ts` не включен.

Сейчас проект использует стандартную навигацию Next.js App Router и анимации внутри компонентов (например, `motion/react` в media-компонентах).

## Почему так

Текущий UI уже имеет локальные анимации (карточки/модальные превью), а глобальные page transitions не являются обязательным требованием для production-стабильности.

## Если решим включить позже

Минимальный план:

1. Определить целевой API (`react` ViewTransition или отдельная библиотека).
2. Включить/настроить поддержку в `next.config.ts` при необходимости.
3. Добавить анимации на уровне маршрутов и проверить fallback для неподдерживаемых браузеров.
4. Прогнать a11y/perf регрессию и обновить этот документ с конкретной реализацией.
